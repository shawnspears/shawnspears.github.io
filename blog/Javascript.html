Phase 0: Week 7

Accio Loop!!

November 16nd, 2015

This will be a short one you guys, I just thought that you might want to learn about some of the distinct differences in looping between Ruby and JavaScript. Or perhaps you're just studying for your Charms class and need to loop through your spells really quickly. Obviously my targeted audience is either/or.

So, let's say that you have the following list of spells:

        var spells = ["Wingardium Leviosa", "Accio", "Stupefy", "Expecto Patronum", "Obliviate"]

Theoretically we know that this is a JavaScript array as it defines the variable spells and assigns it to an array. The array in Ruby would look exactly the same except you could just drop the "var" as it isn't necessary for Ruby syntax. Now that we've gotten that minute detail out of the way, let's go ahead and loop through the data. In Ruby, you can iterate through data using the array.each method, so I've included an example of that as well, however if we're talking about looping then perhaps a while loop would be more relevant here.

Ruby #1: #.each method 

          spells.each do |spell|
            p spell
          end

The #.each method makes it incredibly easy to print out arrays, as you're literally just saying to print each value. Obviously if you just wanted to print an array this would be much more intuitive than looping, but we're talking about looping so we should probably explain how this would work in a while loop.

Ruby #2: While loop 

          i = 0
          while i < spells.length
            p spells[i]
            i += 1
          end

Here we have to do a bit more work for this to work: we have to set a counter to 0 (which will behave as our index number), and print each spell in the array starting with index 0. Because the condition says that the loop will run while i is less than spells.length, the loop will end once it prints the final index. What will this look like in JavaScript?

JavaScript: For loop

          for(var i = 0; i < spells.length; i++) {
            console.log(spells[i])
          }

Essentially what JavaScript syntax does for you, is make the counter mechanism separate from the code block that you want executed each loop. The for loop provides three spaces for different statements. The first statement (var i = 0), sets up the counter variable to an indicated value. Here 0 works because we're using it as an index. The second statement (i < spells.length) is exactly what you would indicate in a while loop. It tells the for loop to run while that condition is being met. The last condition (i++), increments the counter by the indicated value each time it loops; i++ is simply JavaScript syntax for i += 1, which we were familiar with in Ruby.

Hope that helped bring some insight to how looping syntax differs in Ruby vs. JavaScript. Until next time, Muggles!
