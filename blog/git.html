Phase 0: Week 1

The Command Line and Git/GitHub

October 5th, 2015

Okay, if I'm being entirely honest here I was pretty terrified about this week of Phase 0 considering I had absolutely no experience in either using the command line in Terminal, or using Git amd GitHub. You guys, it's literally Week 1 of Phase 0 and my preconceived notion of using Terminal was that it allowed you to relive the experience of using a computer in the 80's and that GitHub was some elite group of wizard developers who knew how to utilize this magic "Git" power while simultanouesly riding on unicorns.

Yeah, to say I was a tad intimidated was a definite understatement.

What I learned this week, however, is that these are both tools that are here to make me the most efficient developer that I can possibly be. There are so many reasons why using the command line and Git are so ridiculously invaluable as a developer: I would personally sum up the reasons as effienciency and collaboration, respectively.

Essentially the command line interface (CLI) allows you to execute commands to the operating system sans the nice graphical user interfact (GUI) that I am so used to. I was born in the new generation, and thus for as long as I can remember navigating through my computer was always a very visual experience. Trying to let go of that intially while learning how to utilize the command line was diffult at first, but repition proved to be essential in getting over this. Learning how to properly use Git required basic command line skills. Git is a version control system that allows a developer to track changes in their code as well as maintains different versions of their code for later access. Developers use version control in order to more efficiently work together to create a project. The fact that version control allows for history tracking and the ability to access previous versions of a project makes it possible for developers to reach a final product while minimizing the amount of mistakes occurring. The efficiency is also maximized through branches that allows a team to work on different features for a project simultaneously while merging to a master branch that should theoretically good to go since its been reviewed (pulled) by other team members. Why use GitHub then? GitHub is online instead of being remote and local, thus making everyone's life easier!
