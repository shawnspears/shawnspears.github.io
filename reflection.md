## What did you learn about CSS padding, borders, and margin by doing this challenge?

  
Learned a lot about all of the above topics simply by experimenting with different values associated with the aforementioned properties. I suppose the main point however, is that padding and margin just refers to the space between the border and the content inside the element (padding), and between the border and the next element (margin). Another important lesson learned is that padding/margin will be applied to the children of the element as well, so the spacing could be thrown off fairly easily unless you indicate otherwise. Fun stuff.


## What did you learn about CSS positioning?


Oh man. Positioning was even more fun than padding/margin/borders. At the very least padding and margins somewhat imply their use in their name. Positioning values, on the other hand, are rather finicky, and don't exactly describe what they're going to do? Well, kind of, at least: the two to watch out for are relative and absolute. Absolute positioning is relative (THIS MAKES ME THINK RELATIVE POSITIONING?) to the containing element (or parent element), and relative positioning is relative (I suppose this actually works here) to where it would have been in normal flow. I learned that it honestly takes a few minutes of playing with positioning in Chrome's dev tools before making a change to the css page!

## What did you learn about adding and formatting elements with CSS in this challenge?

  
I learned that it is incredibly easy to become unorganized in the css page as you add more elements, ids, classes, and so forth. I realized that semantic elements are my best friend because it makes formatting and organization way easier than it would have been with 50 different class names for a div. I also realized that formatting definitely makes a lot of experimentation, and that I need to realize that there are a hundred different ways to get an element formatted the way I want to, and that I shouldn't stress out about trying to do it a specific way (unless there is a standard method of course!).
